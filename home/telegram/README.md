# Telegram-мост для Hermes

Тонкий бот: сообщение из Telegram → Claude Code в доме агента (`claude -p`) →
ответ в чат. Бот не дублирует логику — он зовёт того же Hermes, что и в терминале,
поэтому в Telegram доступны личность, навыки, память и workspace-файлы.

> Обычно этот шаг настраивает за тебя агент по `INSTALL.md`. Ниже — вручную.

## Что нужно
- Дом агента уже стоит (папка с `SOUL.md`, `skills/`, `workspace/`). Эта папка
  `telegram/` лежит внутри него.
- На этой машине установлен и авторизован Claude Code (подписка или API-ключ).
- Машина включена постоянно (для 24/7 — сервер или всегда-включённый комп).

## Запуск
```bash
cd <дом-агента>/telegram
pip install -r requirements.txt
cp .env.example .env      # и заполни: токен, свой user id, доступ к модели
python bot.py
```

## Держать включённым (сервер)
Чтобы бот не падал при выходе из SSH — заверни в `systemd`, `pm2` или хотя бы
`nohup python bot.py &`. Агент по `INSTALL.md` поможет настроить автозапуск.

## Команды
- обычное сообщение — разговор с ассистентом;
- `/reset` — забыть нить последнего разговора (файлы памяти остаются);
- `/start` — приветствие.

Доступ к модели и приватность — см. `.env.example` и корневой `INSTALL.md`.
