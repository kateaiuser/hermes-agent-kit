# INSTALL — для ИИ-агента, который ставит Hermes человеку

> Этот файл читает **Claude (или другой ИИ-агент)**, которого человек попросил
> «поставь мне Hermes». Человек, скорее всего, не технарь. Твоя задача — провести
> установку за него: делай сам всё, что можешь, объясняй простыми словами,
> спрашивай только то, что нельзя сделать без него (регистрация, оплата, логин,
> токен, выбор).

Цель по умолчанию: **сервер 24/7 + подписка Claude** (Telegram отвечает с
телефона всегда, отдельной оплаты за токены нет). Это рекомендованный путь, иди по
нему, если человек не попросил иначе.

## Как себя вести

- Тепло, коротко, на языке человека. Без жаргона. Один шаг → результат → дальше.
- Делай работу сам по SSH: команды, файлы, проверка вывода. Не диктуй человеку
  команды для копипасты, если можешь выполнить их сам.
- Регистрацию, оплату и вход в аккаунты делает человек — объясняй, что выбрать.
- Перед тратой денег и установкой ПО — короткое «ок?».
- В конце доведи до рабочего ассистента **в Telegram** и покажи, что отвечает.

## Шаг 0. Что понадобится (проговори с человеком)

1. **Сервер (VPS), 24/7.** Самый маленький с **2+ ГБ памяти**. Рекомендация:
   - **Hetzner CX22** — 2 vCPU / 4 ГБ, ~€4.5/мес. Лучшее по цене, бери это.
   - Если в его стране с регистрацией Hetzner сложно — **DigitalOcean** (дроплет
     2 ГБ, ~$12/мес), самый дружелюбный интерфейс.
   ОС: **Ubuntu 22.04/24.04**.

2. **Подписка Claude Pro** — этого достаточно для личного ассистента. На ней
   ассистент и работает, **API-ключ не нужен**. Модель по умолчанию — Sonnet, она
   щадит лимиты. **Max не нужен**, если человек не гоняет Claude Code весь день для
   другого; для одного ассистента Pro хватает (упрётся в лимит — апнется потом).
   Запасной путь через API-ключ — в конце, только если подписки не будет.

3. **Аккаунт Telegram** (на телефоне).

> Хочет сперва бесплатно пощупать на своём компьютере, без сервера? — короткая
> врезка в конце файла. Но основной и рекомендованный путь — сервер.

## Шаг 1. Поднять сервер

Помоги человеку зарегистрироваться у провайдера и создать сервер (Ubuntu, тариф
из Шага 0). Регистрация и оплата — он сам. На выходе нужны: **IP-адрес**,
пользователь и **SSH-доступ**. Зайди на сервер:
```bash
ssh <user>@<server-ip>
```

## Шаг 2. Поставить мотор (Claude Code)

На сервере:
```bash
sudo apt update && sudo apt install -y python3 python3-pip python3-venv curl
# Node.js 20 LTS (для Claude Code):
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
npm install -g @anthropic-ai/claude-code
claude --version   # проверка
```

## Шаг 3. Войти подпиской Claude (без API-ключа)

На сервере браузера нет, поэтому используется длинный токен подписки:

1. На **компьютере человека** (там, где есть браузер) выполни:
   ```bash
   claude setup-token
   ```
   Откроется вход в аккаунт Claude — человек логинится своей подпиской. Команда
   выдаст **токен** (живёт ~год). Скопируй его.
2. Этот токен положим в `.env` Telegram-моста на Шаге 5 (переменная
   `CLAUDE_CODE_OAUTH_TOKEN`). Мост передаёт его Claude Code — отдельно за токены
   ничего не списывается, всё в рамках подписки.

> Если человек принципиально без подписки — вместо токена будет `ANTHROPIC_API_KEY`
> (оплата по использованию). Всё остальное идентично.

## Шаг 4. Дом агента

```bash
sudo mkdir -p /srv/hermes/workspace && sudo chown -R $USER /srv/hermes
```
Скопируй сюда содержимое папки `home/` из репозитория (через `git clone` на
сервере, `scp` или `rsync`). Проверь, что на месте `SOUL.md`, `skills/`,
`workspace/`, `telegram/`.

Персонализация:
- **Имя ассистента и обращение НЕ трогай** — ассистент сам спросит человека при
  первом сообщении и подставит в `SOUL.md` (см. `home/CLAUDE.md`, «Первый запуск»).
- `memories/USER.md` можно оставить шаблоном: ассистент наполнит его по ходу
  общения. Если человек хочет рассказать о себе сразу — можешь помочь вписать.

## Шаг 5. Telegram (основной способ общения)

**5.1.** Человек создаёт бота: в Telegram @BotFather → `/newbot` → имя → получает
**токен**. И узнаёт свой **user id** у @userinfobot.

**5.2.** Настрой мост на сервере:
```bash
cd /srv/hermes/workspace/telegram
python3 -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```
Впиши в `.env`:
- `TELEGRAM_BOT_TOKEN` — токен от BotFather;
- `ALLOWED_USER_IDS` — его user id;
- `CLAUDE_CODE_OAUTH_TOKEN` — токен подписки из Шага 3.
  (Или `ANTHROPIC_API_KEY`, если выбрали API.)

**5.3.** Проверь руками: `python bot.py`, напиши боту в Telegram. Работает —
останови (Ctrl+C) и поставь на автозапуск, чтобы жил после выхода из SSH:

```ini
# /etc/systemd/system/hermes-bot.service
[Unit]
Description=Hermes Telegram bridge
After=network-online.target

[Service]
WorkingDirectory=/srv/hermes/workspace/telegram
EnvironmentFile=/srv/hermes/workspace/telegram/.env
ExecStart=/srv/hermes/workspace/telegram/.venv/bin/python bot.py
Restart=on-failure
User=<user>

[Install]
WantedBy=multi-user.target
```
```bash
sudo systemctl daemon-reload && sudo systemctl enable --now hermes-bot
sudo systemctl status hermes-bot     # должен быть active (running)
```

## Шаг 6. (Опционально) Авто-сводки по расписанию

Если человек хочет, чтобы утренний бриф приходил сам, добавь `cron` на сервере:
```bash
# будни в 8:00, результат в файл брифов
0 8 * * 1-5 cd /srv/hermes/workspace && claude -p "Сделай утренний бриф по skills/daily-brief" --permission-mode acceptEdits >> /srv/hermes/workspace/workspace/daily_briefs/$(date +\%F).md 2>&1
```
Делай только по запросу.

---

## Проверка (обязательно)

1. В Telegram: «привет» и «сделай утренний бриф» → отвечает, тон верный (тёплый,
   коротко, ≤3 приоритета).
2. «запомни: проверить, что инбокс работает» → строка появилась в
   `/srv/hermes/workspace/workspace/inbox.md`.
3. Граница: без «да» ничего не отправляет/не публикует/не удаляет.
4. Перезагрузи сервер (`sudo reboot`) → бот сам поднялся (systemd), Telegram снова
   отвечает.
5. Если флаги `claude` в этой версии отличаются и бот ругается — поправь вызов в
   `telegram/bot.py` (функция `ask_claude`) под текущий `claude --help`.
6. Скажи человеку главное: **«Пиши ему в Telegram как другу. Остальное он делает сам.»**

---

## Если что-то пошло не так

- `claude: command not found` → Node/Claude Code не встали (Шаг 2).
- Ошибка авторизации → проверь `CLAUDE_CODE_OAUTH_TOKEN` (не истёк ли, без лишних
  пробелов) или `ANTHROPIC_API_KEY` и баланс.
- Бот молчит → `systemctl status hermes-bot`; верный ли `ALLOWED_USER_IDS`; жив ли
  токен бота.
- Мало памяти / падает → бери тариф с 2+ ГБ (см. Шаг 0).
- «Звучит как бот», сахарит → `SOUL.md`, секция запретов, усиль под вкус человека.

## Врезка: попробовать на своём компьютере (без сервера)

Если хочется сначала бесплатно пощупать: установи Claude Code локально (Node 18+,
`npm install -g @anthropic-ai/claude-code`, вход `claude` под подпиской), сделай
дом командой `mkdir -p ~/hermes && cp -R home/. ~/hermes/`, запусти
`cd ~/hermes && claude` и поговори в терминале — ассистент сам спросит, как его
называть. Telegram можно поднять и тут (Шаг 5, пути `~/hermes/...`),
но он отвечает, только пока компьютер включён. Понравилось — переезжай на сервер.
