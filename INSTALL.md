# INSTALL — для AI-агента, который ставит ассистента

> Этот файл читает **AI-агент** (Claude Code / Cowork, сам Hermes или подобный,
> умеющий выполнять команды), которого человек попросил «подготовь / поставь мне
> ассистента». Человек **не технарь** — сделай за него всё, что можешь сам.
>
> Если ты агент без доступа к терминалу (например, Claude в браузере) — ты не
> сможешь всё выполнить; тогда проведи человека по шагам и попроси его агента с
> терминалом (Claude Code / Cowork) или сам Hermes доделать техническую часть.

**Цель по умолчанию:** поставить ассистента на **сервер 24/7** (чтобы Telegram
отвечал всегда), на бесплатной модели Nous Portal для старта.

## Разделение труда

**Делаешь сам (агент):** всё техническое — установка Hermes на сервер,
распаковка и копирование `soul/`, настройка Telegram-шлюза, автозапуск, проверка.

**Просишь у человека (только это, нельзя за него):**
1. завести и оплатить маленький VPS (дай выбор, объясни);
2. один раз войти в модель в браузере (Nous Portal, можно бесплатно);
3. создать Telegram-бота у @BotFather и прислать тебе токен + свой id (@userinfobot).

---

## Шаг 1. Сервер (VPS)

Помоги человеку завести дешёвый VPS. Рекомендация:
- **Hetzner CX22** — 2 vCPU / 4 ГБ, ~€4.5/мес. Лучшее по цене.
- **DigitalOcean** (~$12/мес) — если с регистрацией Hetzner сложно.
ОС: **Ubuntu 22.04/24.04**. Регистрация и оплата — на человеке; объясни, что
выбрать. Получи SSH-доступ (IP + пользователь) и зайди:
```bash
ssh <user>@<server-ip>
```

## Шаг 2. Поставить Hermes (движок) на сервер

```bash
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
hermes --version   # проверка
```

## Шаг 3. Подключить модель

Запусти мастер Hermes для входа к провайдеру. Человек логинится в **Nous Portal**
(вход в браузере; на сервере — по коду/токену, мастер подскажет). На бесплатном
тарифе выбери бесплатную модель — для старта хватит. (Claude — Шаг 6.)

## Шаг 4. Вложить характер (soul) — делаешь сам

У тебя уже есть этот набор (ты его скачал/склонировал). Скопируй содержимое `soul/`
в дом Hermes `~/.hermes/` на сервере. Человеку **руками ничего копировать не нужно**:
```bash
cp soul/SOUL.md ~/.hermes/SOUL.md
cp -R soul/skills/assistant ~/.hermes/skills/assistant
```

## Шаг 5. Telegram

Попроси человека прислать: токен бота (@BotFather → /newbot) и его числовой id
(@userinfobot). Затем на сервере:
```bash
hermes gateway setup     # выбери Telegram, вставь токен и id человека
hermes gateway start
```
Поставь шлюз на **автозапуск**, чтобы бот жил после перезагрузки (systemd-сервис,
запускающий `hermes gateway start`; проверь, что после `reboot` поднимается сам).

## Шаг 6. (По желанию) Claude вместо бесплатной модели

В чате с ботом: `/model` → **Claude Sonnet 4.6** (баланс) или **Opus 4.8**
(максимум). Платно (через Nous Portal или свой `ANTHROPIC_API_KEY`). Делай только
по желанию человека.

---

## Проверка

1. Человек пишет боту в Telegram «привет» → отвечает, тон тёплый, спрашивает, как
   его называть.
2. «запомни: проверить память» → позже вспоминает.
3. `reboot` сервера → бот сам поднялся.
4. Без «да» человека ничего не отправляет/не публикует/не удаляет.

## Если что-то не так

- Бот молчит → запущен ли `hermes gateway start`; верный ли id; жив ли токен.
- Ошибка модели → проверь вход в Nous Portal (или тариф/баланс для Claude).
- Мало памяти/падает → тариф с 2+ ГБ.

## Быстро попробовать без сервера (необязательно)

Если человек хочет сперва бесплатно пощупать на своём компьютере: поставь
приложение Hermes локально (Mac `.dmg` / Windows `.exe` с
hermes-agent.nousresearch.com), проделай Шаги 3–5 локально, `soul/` скопируй в
локальный `~/.hermes/`. Минус: бот отвечает, только пока компьютер включён —
для «всегда» нужен сервер.
